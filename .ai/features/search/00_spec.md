# 00_spec - search

작성: Codex
일시: 2026-05-28

## 기능 목표
- 기존 시장 대시보드를 첫 번째 메인 탭으로 유지하고, 사용자가 설정 탭에서 한국/미국 주식과 ETF를 검색해 개별 종목 탭으로 추가할 수 있게 한다.
- 추가된 종목 탭에서는 해당 종목의 상세 시세 정보와 가격 차트를 제공하며, 설정 탭은 항상 탭 목록의 마지막에 위치한다.

## 기능명
- feature_name: search
- naming_reason: 하네스에서 `feature_name_locked: true`로 제공한 기능명이므로 변경하지 않는다. 기능의 핵심 진입점도 종목 검색이다.

## 구체적 요구사항
- 앱 상단 또는 기존 헤더 영역에 탭 내비게이션을 추가한다.
- 기본 탭 순서는 `대시보드`, 사용자가 추가한 종목 탭들, `설정` 순서여야 한다.
- `대시보드` 탭은 기존 대시보드 화면과 데이터를 그대로 표시하는 기본 탭이다.
- `설정` 탭은 항상 가장 마지막에 표시되어야 하며, 사용자가 종목을 추가하거나 삭제해도 마지막 위치가 유지되어야 한다.
- 설정 탭에는 종목 검색 입력이 있어야 한다.
- 검색 대상은 한국/미국 상장 주식과 ETF로 제한한다.
- 검색 결과에는 최소한 종목명, 표시 심볼, 거래소/시장, 국가, 상품 유형(주식 또는 ETF), 통화가 표시되어야 한다.
- 검색 결과에서 주식/ETF가 아닌 항목(지수, 환율, 암호화폐, 선물, 옵션, 펀드 등)은 추가할 수 없어야 한다.
- 검색어가 비어 있거나 너무 짧으면 요청하지 않고 안내 상태를 표시한다. 기본값은 공백 제거 후 1자 이상부터 검색 가능으로 한다.
- 검색 결과가 없거나 외부 데이터 제공자가 실패하면 안전한 오류/빈 상태를 표시하고 앱 전체가 깨지지 않아야 한다.
- 검색 결과의 `추가` 동작은 해당 종목 전용 탭을 생성한다.
- 이미 추가된 종목을 다시 추가하면 중복 탭을 만들지 않고 기존 종목 탭으로 이동한다.
- 종목 탭의 라벨은 미국 종목은 `QLD`처럼 티커를 대문자로 표시하고, 한국 종목은 사용자가 식별 가능한 표시 심볼을 사용한다.
- 사용자가 종목을 추가하면 새 종목 탭을 즉시 활성화한다.
- 추가된 종목 탭 목록은 브라우저 `localStorage`에 저장해 새로고침 후에도 유지한다.
- 설정 탭에서는 추가된 종목 목록을 확인하고 필요하면 제거할 수 있어야 한다.
- 종목 탭에서 제거된 종목은 탭 목록과 `localStorage`에서 삭제된다.
- 종목 탭은 해당 종목의 상세 정보를 표시한다. 최소 표시 항목은 현재가, 전일 대비, 등락률, 시가, 고가, 저가, 종가 또는 전일 종가, 거래량, 통화, 거래소, 시장 상태, 기준 시각, 데이터 소스이다.
- 제공자 응답에서 일부 상세 항목이 누락되면 `-` 또는 결측 상태로 표시하고, 숫자 데이터의 NaN/Infinity는 API 경계에서 `null`로 정규화한다.
- 종목 탭에는 가격 차트를 표시한다.
- 차트 기본 기간은 최근 1개월이며, 기본 기간 선택지는 `1D`, `1M`, `3M`, `1Y`로 한다.
- 차트는 외부 차트 라이브러리를 새로 추가하지 않고 React/SVG 또는 HTML/CSS 기반의 가벼운 자체 렌더링으로 구현하는 것을 기본값으로 한다.
- 상세/차트 데이터 로딩 중, 실패, 부분 결측 상태를 각각 구분해서 표시한다.
- 상승, 하락, 보합, 결측 상태는 기존 대시보드와 같은 색상 의미를 유지한다.
- 백엔드에는 검색 API와 종목 상세/차트 API를 추가한다.
- 외부 데이터 제공자 호출은 기존 어댑터 계층 뒤에서 처리해 FastAPI 라우터와 비즈니스 로직이 직접 외부 API 세부사항에 의존하지 않게 한다.
- 외부 데이터 제공자 장애는 안전한 JSON 오류 응답으로 변환한다.
- 검색/상세/차트 API 테스트는 정상 경로, 비정상 입력, 제공자 누락 응답, 제공자 예외 경로를 포함한다.
- 프론트엔드 변경은 기존 Vite/React 구조 안에서 구현하고, 새 테스트가 필요하면 루트 `tests/` 하위에 둔다.

## 이번 범위에서 제외하는 것
- 사용자 계정 기반 서버 저장소 또는 로그인 기능은 제외한다. 현재 앱에 인증/사용자 저장 구조가 없으므로 브라우저 로컬 저장을 기본값으로 한다.
- 매수/매도, 포트폴리오 수익률, 알림, 관심종목 그룹화, 자동 매매 기능은 제외한다. 이번 요청은 검색 후 종목 탭 추가와 상세 조회에 집중한다.
- 한국어 종목명 완전 자동 보정이나 증권사 수준의 심볼 정규화는 제외한다. 외부 데이터 제공자가 반환하는 검색 결과를 기준으로 검증한다.
- 실시간 스트리밍 시세는 제외한다. 기존 구조와 동일하게 요청 시점의 지연/최근 데이터를 조회한다.
- 새 유료 데이터 제공자나 API 키가 필요한 연동은 제외한다. 기존 Yahoo Finance 기반 어댑터 확장을 기본값으로 한다.
- 차트용 대형 시각화 라이브러리 도입은 제외한다. 현재 프론트엔드 의존성이 매우 작고, 새 외부 의존성은 위험도를 높이기 때문이다.

## 입력과 출력
- 검색 입력:
  - 타입: 문자열
  - 범위: 공백 제거 후 1자 이상, 과도한 외부 호출 방지를 위해 서버에서 최대 길이 50자로 제한
  - 예: `qld`, `AAPL`, `삼성전자`, `005930`
- 검색 API 출력:
  - `results`: 검색 결과 배열
  - 각 항목: `symbol`, `displaySymbol`, `name`, `exchange`, `country`, `instrumentType`, `currency`, `providerSymbol`, `source`
  - 허용 `instrumentType`: `stock`, `etf`
- 종목 탭 입력:
  - 검색 결과에서 선택한 종목 객체 또는 이를 재조회할 수 있는 안정적인 `providerSymbol`
  - 탭 저장 데이터는 최소 `symbol`, `displaySymbol`, `name`, `exchange`, `country`, `instrumentType`, `currency`, `providerSymbol`를 포함한다.
- 상세 API 출력:
  - `instrument`: 종목 식별 정보
  - `quote`: 현재가, 전일 대비, 등락률, 시가, 고가, 저가, 종가/전일 종가, 거래량, 통화, 시장 상태, 기준 시각, 데이터 소스
  - `chart`: 선택 기간의 시계열 포인트 배열. 각 포인트는 `timestamp`, `open`, `high`, `low`, `close`, `volume`을 포함하며 누락 값은 `null`이다.
- 비정상 입력 처리:
  - 빈 검색어 또는 길이 초과는 400 JSON 오류로 응답한다.
  - 허용되지 않는 상품 유형은 검색 결과에서 제외하고, 상세 요청 시 확인되면 400 또는 404 JSON 오류로 응답한다.
  - 제공자 장애는 502 또는 기존 정책에 맞춘 안전한 JSON 오류로 응답하며 내부 예외 메시지는 노출하지 않는다.

## 기존 코드 영향
- 수정 예상 파일:
  - `src/frontend/src/App.jsx`: 탭 상태, 대시보드/종목/설정 화면 전환, 로컬 저장 상태 관리
  - `src/frontend/src/index.css`: 탭, 설정 검색 UI, 종목 상세/차트 반응형 스타일
  - `src/frontend/src/components/`: 설정 검색, 종목 탭 상세, 차트, 탭 내비게이션 컴포넌트 추가 가능
  - `src/backend/main.py`: 검색 및 종목 상세 API 엔드포인트 추가
  - `src/backend/app/adapters/base.py`: 검색/상세 조회 인터페이스 확장 또는 별도 어댑터 인터페이스 추가
  - `src/backend/app/adapters/yahoo_adapter.py`: Yahoo 검색/차트 응답 파싱, 한국/미국 주식/ETF 필터링, 캐시 적용
  - `src/backend/app/adapters/mock_adapter.py`: 테스트 가능한 검색/상세 mock 데이터 추가
  - `src/backend/app/schemas/market.py`: 검색 결과, 종목 상세, 차트 포인트 스키마 추가
  - `tests/backend/`: API, 스키마, 어댑터 테스트 추가/확장
- 사용할 기존 모듈/유틸:
  - 기존 `YahooMarketDataAdapter`의 HTTP 요청, 타임아웃, 캐시, 유한 숫자 정규화 패턴
  - 기존 Pydantic 스키마의 `extra="forbid"`와 숫자 정규화 정책
  - 기존 `MarketItemCard`의 숫자/등락 표시 의미와 색상 톤
- 충돌 가능성:
  - 현재 `MarketDataAdapter`는 `get_dashboard_data`만 추상 메서드로 갖고 있어 검색/상세 기능 확장 방식 결정이 필요하다.
  - 프론트엔드가 현재 단일 화면 구조라 `App.jsx`가 커질 수 있으므로 컴포넌트 분리가 필요하다.
  - 현재 한글 문자열 일부가 깨져 보이는 파일이 있어 신규 UI 문구 작성 시 인코딩을 UTF-8로 유지해야 한다.

## 기술적 제약
- 프론트엔드는 기존 React 18 + Vite 구조를 유지한다.
- 백엔드는 기존 FastAPI + Pydantic 구조를 유지한다.
- 외부 데이터는 기본적으로 기존 Yahoo Finance 비공식 엔드포인트를 어댑터 뒤에서 사용한다.
- 새 외부 의존성은 기본적으로 추가하지 않는다. 필요성이 명확해지는 경우 다음 단계 계획에서 별도 근거와 위험을 기록한다.
- 외부 호출은 기존 설정의 타임아웃을 따르고, 검색/상세 데이터에도 캐시를 적용한다.
- 일괄 또는 반복 조회는 제한된 동시성과 부분 실패 처리를 유지한다.
- API 응답의 수치 데이터는 내보내기 전에 NaN/Infinity를 `null`로 정규화한다.
- 브라우저 저장 데이터는 파싱 실패 가능성을 처리하고, 깨진 저장값은 무시하거나 초기화한다.
- 모바일과 데스크톱에서 탭 라벨, 검색 결과, 상세 카드, 차트가 겹치지 않아야 한다.

## 기본 결정 기록
- 설정 탭 추가 위치: 항상 마지막. 사용자가 명시했고 탭 추가/삭제 후에도 유지해야 하므로 고정한다.
- 대시보드 위치: 항상 첫 번째. 사용자가 기존 대시보드를 메인 탭으로 요청했으므로 고정한다.
- 추가 종목 저장 방식: `localStorage`. 현재 인증/서버 저장소가 없고, 설정 추가 상태를 새로고침 후 유지하는 것이 자연스럽기 때문이다.
- 중복 추가 처리: 새 탭을 만들지 않고 기존 탭으로 이동. 같은 종목 탭 중복은 사용성을 떨어뜨리고 상태 관리가 복잡해진다.
- 데이터 제공자: 기존 Yahoo 어댑터 확장. 이미 프로젝트에 Yahoo 기반 어댑터와 테스트 패턴이 있어 새 제공자를 도입하지 않는 것이 범위와 위험을 줄인다.
- 차트 구현: 새 라이브러리 없이 가벼운 자체 렌더링. 현재 프론트엔드 의존성이 작고 스펙 단계에서 새 의존성을 확정할 필요가 없다.
- 기본 차트 기간: 1개월. 사용자가 기간을 지정하지 않았고, 종목 상세에서 단기 흐름을 보기에 무난한 기본값이다.
- 검색 최소 길이: 공백 제거 후 1자. 티커 검색은 `Q`, `F`처럼 1자 종목도 가능하므로 2자 이상으로 제한하지 않는다.
