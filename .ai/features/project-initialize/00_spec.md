# 00_spec - project-initialize

작성: Codex
일시: 2026-05-28

## 기능 목표
- React 화면과 FastAPI 백엔드로 주식/시장 대시보드 초기 프로젝트를 구성한다.
- 첫 화면에서 미국 대표지수, 한국 대표지수, 기타국 대표지수, 주요 환율을 구역별로 나누어 한 번에 확인할 수 있게 한다.

## 기능명
- feature_name: project-initialize
- naming_reason: 하네스에서 잠긴 기능명이며, React/FastAPI 기반 프로젝트 초기 구성을 만드는 범위와 일치한다.

## 구체적 요구사항
- 프론트엔드는 React 기반 단일 화면 대시보드로 구성한다.
- 백엔드는 FastAPI 기반 API 서버로 구성한다.
- 프로덕션 코드는 프로젝트 계약에 따라 루트 `src/` 하위에 둔다.
- 첫 화면은 다음 4개 섹션을 명확히 구분해 표시한다.
- 미국 대표지수: S&P 500, Nasdaq Composite, Dow Jones Industrial Average, Russell 2000을 기본 표시 대상으로 한다.
- 한국 대표지수: KOSPI, KOSDAQ, KOSPI 200을 기본 표시 대상으로 한다.
- 기타국 대표지수: Nikkei 225, Shanghai Composite, Hang Seng, Euro Stoxx 50, FTSE 100, DAX를 기본 표시 대상으로 한다.
- 환율: USD/KRW, EUR/KRW, JPY/KRW, CNY/KRW, USD/JPY를 기본 표시 대상으로 한다. JPY/KRW는 금융권 표기 관례를 고려해 100엔 기준 표기가 필요하면 라벨에 단위를 명시한다.
- 각 시장 항목은 최소한 이름, 심볼 또는 코드, 현재값, 등락값, 등락률, 기준 시각을 표시한다.
- 데이터 로딩 중, 데이터 없음, API 실패 상태를 화면에서 구분해 보여준다.
- 백엔드는 대시보드 초기 화면에 필요한 데이터를 한 번에 받을 수 있는 집계 API를 제공한다.
- 외부 데이터 제공자 연동은 프로젝트 계약에 따라 어댑터 인터페이스 뒤에 둔다.
- 초기 구현에서 실제 API 키나 유료 데이터 공급자는 요구하지 않는다. 자격 증명 없이 가능한 공개 데이터 또는 개발용 목 데이터 어댑터를 기본값으로 사용한다.
- 수치 응답은 NaN, Infinity 같은 비표준 값을 JSON으로 내보내기 전에 null 등 안전한 표준 타입으로 정규화한다.
- 등락률은 백분율 단위임을 명확히 처리하고, 비율값과 백분율값의 중복 변환을 방지한다.

## 이번 범위에서 제외하는 것
- 사용자 인증, 개인 포트폴리오, 관심종목 저장은 제외한다. 초기 대시보드 범위를 넘고 별도 데이터 모델이 필요하다.
- 실시간 스트리밍 시세는 제외한다. 초기 버전은 HTTP 조회와 새로고침 중심으로 정의한다.
- 매매 주문, 알림, 차트 드릴다운, 뉴스, 기업 상세 페이지는 제외한다. 핵심 지수/환율 현황판 이후 확장 기능으로 둔다.
- 유료 API 키 또는 사용자별 비밀값 설정은 제외한다. 현재 단계에서 사용할 수 있는 자격 증명이 없기 때문이다.
- 모바일 앱 또는 데스크톱 앱 패키징은 제외한다. 웹 대시보드 구현이 우선이다.

## 입력과 출력
- 프론트엔드 입력: 초기 화면 진입, 수동 새로고침 버튼 클릭, 필요 시 백엔드 API 호출 결과.
- 백엔드 입력: 집계 API 요청. 초기 버전은 필수 쿼리 파라미터 없이 기본 지수/환율 목록을 반환한다.
- 백엔드 출력: 섹션별 시장 데이터 배열을 포함한 JSON.
- 권장 JSON 구조: `generatedAt`, `sections`, `items`, `name`, `symbol`, `value`, `change`, `changePercent`, `currency`, `marketStatus`, `asOf`, `source`.
- 비정상 입력 처리: 알 수 없는 섹션이나 심볼 요청이 추가될 경우 400 계열 오류와 설명 메시지를 반환한다.
- 외부 데이터 실패 처리: 백엔드는 실패한 제공자와 사유를 로깅 가능한 오류로 분리하고, 프론트엔드에는 섹션 또는 항목 단위 오류 상태를 반환한다.
- 프론트엔드 출력: 카드/표 형태의 섹션별 대시보드, 등락 색상 표시, 기준 시각, 로딩/오류/빈 상태 UI.

## 기존 코드 영향
- 현재 저장소에는 실제 앱 구현이 없고 `src/source_code_root` 자리표시자만 확인된다.
- 다음 단계에서 `src/` 하위에 프론트엔드와 백엔드 구조를 새로 계획해야 한다.
- 기존 테스트 파일은 확인되지 않았다. 테스트가 추가되면 프로젝트 계약에 따라 루트 `tests/` 하위에 둔다.
- 기존 기능과의 직접 충돌 가능성은 낮지만, 새 프로젝트 골격과 외부 의존성 추가 가능성이 있다.
- `.ai/`, `presets/`, 하네스 관련 파일은 이번 기능 구현 대상이 아니며 산출물 기록 외에는 수정하지 않는다.

## 기술적 제약
- 프론트엔드 프레임워크는 React를 사용한다.
- 백엔드 프레임워크는 FastAPI를 사용한다.
- 외부 데이터 공급자는 어댑터 인터페이스 뒤로 분리한다.
- 새 의존성 파일이나 실행 스크립트가 필요하면 프로젝트 계약의 경로 원칙을 지켜 `src/` 또는 `tests/` 하위에 둔다.
- 화면은 한 번에 주요 섹션이 보이도록 정보 밀도를 적절히 유지하고, 운영형 대시보드에 맞는 절제된 UI를 사용한다.
- 네트워크, 파일, 프로세스 실행 결과는 실패 가능성을 명시적으로 처리한다.
- 수치 데이터는 JSON 직렬화 전에 안전한 표준 타입으로 정규화한다.
- 브라우저 호환성은 최신 Chromium 계열을 기본 검증 대상으로 한다.

## 기본 결정 및 근거
- 기본 결정: `feature_name`은 `project-initialize`로 유지한다.
- 근거: 하네스에서 `feature_name_locked: true`로 제공했다.
- 기본 결정: 초기 데이터 소스는 API 키가 필요 없는 공개 데이터 또는 목 데이터 어댑터로 시작한다.
- 근거: 현재 자격 증명이 제공되지 않았고, 외부 제공자는 교체 가능한 어댑터 뒤에 둬야 한다.
- 기본 결정: 대시보드는 4개 섹션을 단일 화면에 배치한다.
- 근거: 사용자가 "한번에 보였으면" 좋겠다고 요청했다.
- 기본 결정: 실시간 스트리밍은 제외하고 HTTP 조회/새로고침을 우선한다.
- 근거: 초기 프로젝트 구성 범위에서 구현 위험과 외부 데이터 제약을 낮춘다.
- 기본 결정: 위험도는 `high`로 판단한다.
- 근거: React/FastAPI 새 프로젝트 구조와 새 외부 의존성 가능성이 있고 사용자 경험에 직접 영향을 주는 화면 정책이 포함된다.
